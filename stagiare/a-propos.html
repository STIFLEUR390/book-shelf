Comment l'application a été développée

Cette page explique étape par étape comment créer une application web moderne pour afficher une collection de livres à partir de l'API publique OpenLibrary, d'abord en JavaScript natif puis en Vue.js. Idéal pour les débutants souhaitant apprendre à manipuler les API, le DOM et les frameworks modernes.

1. Version JavaScript natif (Vanilla JS)

  1. Création de la structure HTML et du design avec Bootstrap 5 :
    On commence par réaliser une page HTML avec une barre de recherche, des filtres (type, année), une grille responsive pour afficher les livres, et une pagination. Bootstrap permet de rendre l'interface moderne et responsive facilement.
    Documentation Bootstrap
  2. Ajout du JavaScript :
    On écrit un fichier JS qui :
    • Récupère les éléments du DOM (formulaire, filtres, grille, pagination...)
    • Gère la recherche et les filtres en écoutant les événements
    • Fait des requêtes à l'API OpenLibrary pour obtenir les livres selon les critères
    • Affiche dynamiquement les livres dans la grille (création de cartes HTML via JS)
    • Gère la pagination (calcul du nombre de pages, navigation...)
    • Affiche les couvertures de livres via l'API Covers d'OpenLibrary, ou une image de remplacement si absente
    API Search OpenLibrary | API Covers OpenLibrary
  3. Conseils pour les débutants :
    • Bien séparer le HTML, le CSS et le JS pour plus de clarté
    • Utiliser fetch pour les appels API (Guide Fetch MDN)
    • Manipuler le DOM avec document.getElementById, addEventListener, etc.

2. Version Vue.js 3 (avec CDN)

  1. Pourquoi passer à Vue.js ?
    Vue.js permet de rendre l'application plus réactive, plus facile à maintenir et à faire évoluer. Il simplifie la gestion des états, des listes et des interactions utilisateur. Documentation Vue.js
  2. Intégration de Vue.js via CDN :
    On ajoute simplement le script CDN de Vue dans le HTML, sans installation complexe. On encapsule l'interface dans un <div id="app"> et on utilise la syntaxe Vue (v-model, v-for, v-if...).
  3. Refonte de la logique :
    Toute la logique (recherche, filtres, pagination, affichage dynamique) est déplacée dans un composant Vue. Les données et méthodes sont déclarées dans data() et methods, et le rendu HTML est automatiquement mis à jour selon l'état de l'application.
  4. Bonnes pratiques :
    • Utiliser v-cloak pour éviter le flash de contenu non stylé
    • Bien structurer les données et méthodes dans Vue
    • Lire la documentation officielle pour comprendre les concepts de base (réactivité, composants, etc.)
